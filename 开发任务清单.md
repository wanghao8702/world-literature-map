# 开发任务清单（带验收标准）—— 世界文学巡礼 3D 文学地图

> 配合 `技术路径与开发交接.md` 与 `literary-map.html` 使用。每个 ISSUE 尽量独立、可单独验收。
> 标注：**P0** 最高优先（还原原片质感/灵魂）｜**P1** 信息量与氛围｜**P2** 交互与多媒体｜**ENG** 工程性。
> 约定：所有改动**不得**把已修复的性能反模式加回（毛玻璃 `backdrop-filter`、`mix-blend-mode`、未限制的渲染像素比）。

---

## 数据字段规范（`books.geojson`）

标准 GeoJSON `FeatureCollection`，每个 `Feature` 为一个 `Point`，书目信息放在 `properties`：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `geometry.coordinates` | `[number, number]` | ✓ | `[经度, 纬度]`，经度在前 |
| `id` | string | ✓ | 稳定唯一 id，用于深链/去重（如 `b001`） |
| `title` | string | ✓ | 书名（中文） |
| `title_en` | string | – | 原名 / 英文名（卡片副标题，斜体） |
| `author` | string | ✓ | 作者 |
| `year` | number | ✓ | 出版/成书年份 |
| `century` | number | – | 世纪分组（用于时间轴/过滤，如 `21`） |
| `country` / `city` | string | ✓/– | 国家 / 城市 |
| `region` | string | ✓ | 地区标签（与筛选胶囊一致：亚洲/欧洲/美洲/中东·非洲…） |
| `rating` | number | – | 豆瓣评分 |
| `themes` | string[] | – | 主题标签，用于主题筛选/连线分组 |
| `blurb` | string | ✓ | 一句话简介（**用自己的话**，勿抄原文/书摘） |
| `quote` | string | – | 名句。**版权注意**：勿直接复制受版权保护的原文；优先公有领域作品或自有改写，否则留空 |
| `cover` | string(URL) | – | 封面图地址，留空时卡片用文字封面兜底 |
| `links` | object | – | `{ douban, buy, read }` 外链，留空即不显示 |
| `tour_order` | number | – | 默认巡游顺序；为空时按数组顺序 |

> 样例文件已含 12 本，`quote/cover/links` 为占位空串，填真实数据时按上表补齐。规模化到 1000+ 本时此文件会很大，配合 **ISSUE-01 懒加载** 与 **ISSUE-02 聚合**。

## ISSUE-01 ｜ENG｜数据层：内联数组 → 外部 GeoJSON

**目标**：把 `literary-map.html` 里内联的 `books` 数组改为运行时加载 `books.geojson`，字段映射到现有的标记/书卡/筛选逻辑。

**实现要点**
- `fetch('books.geojson')` → 解析 `features`，映射 `properties` 到内部书对象（注意 `geometry.coordinates` → `coord`）。
- 顶部"跨越 N 国 · 收录 M 部"改为从加载结果动态计算。
- 加载中显示轻量占位/loading；加载失败有兜底提示。

**验收标准**
- [ ] 删除内联数组后，地图仍能从 `books.geojson` 正确渲染全部标记。
- [ ] 书卡字段（含 `title_en`、评分、国家城市、blurb）显示正确。
- [ ] 地区筛选、巡游仍正常工作。
- [ ] 故意改错文件名时，页面不白屏，有可读的错误提示。

**依赖**：无（其他多数 issue 依赖本条完成）。

---

## ISSUE-02 ｜ENG｜标记聚合（clustering）

**目标**：应对 1000+ 标记，避免一屏数百个 marker 造成卡顿。

**实现要点**
- 用地图引擎内置 cluster 源（GeoJSON source `cluster: true`）或 supercluster。
- 簇显示数量徽标；点击簇放大/展开；低缩放聚合、高缩放散开。
- 印章式单点样式保留；簇用另一种醒目样式。

**验收标准**
- [ ] 载入 1000+ 条测试数据（可程序复制现有数据扩充）时，平移/缩放帧率明显优于不聚合。
- [ ] 点击簇能平滑放大并最终展开为单点。
- [ ] 单点点击仍弹出正确书卡。

**依赖**：ISSUE-01。

## ISSUE-03 ｜P0｜美术风格：可切换"皮肤"+ 接入水墨/AI 贴图

**目标**：把底图从 Esri 物理图换成水墨/古地图质感，并做成可切换皮肤机制，便于试不同贴图。

**实现要点**
- 抽象出"皮肤"配置（底图源 + 调色 + sky/雾 + 暗角强度）。
- 至少接入一张水墨/AI 生成贴图（地理配准的栅格源，或自定义 style）。
- 保留 Esri 作为"写实"皮肤备选。

**验收标准**
- [ ] 提供 ≥2 套皮肤，可一键切换且不刷新页面。
- [ ] 水墨皮肤下整体观感接近"古地图"参考；切换后标记/书卡/巡游不受影响。
- [ ] 切换不引入毛玻璃/混合模式等性能反模式。

**依赖**：无（建议 ISSUE-01 之后）。

---

## ISSUE-04 ｜P0｜时间轴展开

**目标**：底部年份/世纪滑块，拖动时按 `year`/`century` 过滤与渐显标记，可播放"随时间展开"动画。

**实现要点**
- 滑块范围由数据 min/max 年份决定；支持"全部"。
- 标记按是否 ≤ 当前年份显示/隐藏，带淡入；播放按钮自动推进。
- 与地区筛选可叠加生效。

**验收标准**
- [ ] 拖动滑块时标记按年份正确增减。
- [ ] 播放动画平滑，可暂停/重置。
- [ ] 与地区筛选叠加时结果正确（交集）。

**依赖**：ISSUE-01。

---

## ISSUE-05 ｜P0｜弧线连接图层

**目标**：在指定书目/城市之间绘制流动虚线（如丝绸之路、某作家足迹、影响关系）。

**实现要点**
- great-circle / 贝塞尔弧线；动态 dash 形成"流动"效果。
- 连线数据可配置（一组 `[fromId, toId]` 或一条有序路径）。
- 与时间轴/筛选联动（仅显示当前可见节点间的连线）。

**验收标准**
- [ ] 能根据配置绘制至少一条多点弧线路径。
- [ ] 虚线有可见的流动动画且不卡。
- [ ] 节点被时间轴/筛选隐藏时，相关连线同步隐藏。

**依赖**：ISSUE-01（建议在 ISSUE-04 后）。

---

## ISSUE-06 ｜P0｜故事 / 章节模式（还原视频灵魂）

**目标**：像翻书一样，带旁白文字 + 相机自动逐站讲解。

**实现要点**
- 定义"章节"数据：每章 = 标题 + 旁白文本 + 一条相机路径（中心/缩放/俯仰）+ 高亮的书目 id 列表。
- 上一步/下一步导航 + 自动播放；进入章节时相机平滑移动、目标书卡/高亮出现。
- 相机优先用平滑路径曲线而非生硬 `flyTo`（见 ISSUE-08）。

**验收标准**
- [ ] 至少 1 个完整故事（≥3 章）可从头播放到尾。
- [ ] 每章旁白与相机/高亮同步出现，可手动前后翻。
- [ ] 退出故事模式后恢复自由浏览。

**依赖**：ISSUE-01；体验上依赖 ISSUE-08。

## ISSUE-07 ｜ENG｜性能模式开关

**目标**：右上角一键在「3D 地形」与「流畅平面」间切换，照顾弱机。

**实现要点**
- 开：`setTerrain(terrainSource…)`；关：`setTerrain(null)` 并可降低 pitch。
- 记住用户选择（本地/托管环境用 `localStorage`；沙箱预览用内存态）。

**验收标准**
- [ ] 切换即时生效，关闭地形后帧率明显提升。
- [ ] 关闭地形后地图仍可用（标记/书卡/巡游正常）。

**依赖**：无。

---

## ISSUE-08 ｜ENG｜相机：平滑路径曲线 + 开场动画 + idle 自动巡航

**目标**：提升镜头质感。

**实现要点**
- 巡游/章节用相机路径插值（自由相机或多段缓动）替代单次 `flyTo`。
- 开场：从高空/太空俯冲落到地图初始视角。
- 无操作 N 秒后缓慢自转"呼吸"，有操作即停。

**验收标准**
- [ ] 巡游镜头明显比当前更顺、无生硬跳变。
- [ ] 首次加载有一段开场动画。
- [ ] idle 自动巡航可触发且交互即停。

**依赖**：与 ISSUE-06 配合最佳。

---

## ISSUE-09 ｜P2｜搜索 + 深链分享

**目标**：顶部搜索框 + URL hash 定位。

**实现要点**
- 按 `title`/`author`/`country` 搜索，命中后 `flyTo` 并弹卡。
- URL 形如 `#book=b001`，载入时自动定位；打开书卡时更新 hash。

**验收标准**
- [ ] 搜索能命中并飞到目标、弹出书卡。
- [ ] 带 `#book=...` 打开页面会自动定位到该书。
- [ ] `localStorage`/hash 在沙箱限制下有兜底（见交接文档第 6 节）。

**依赖**：ISSUE-01。

---

## ISSUE-10 ｜P1/P2｜氛围与多媒体（可拆分）

**目标**：古地图装饰、会动的装饰元素、声音、晨昏光照等氛围增强。

**子项（可各自独立成 issue）**
- 罗盘玫瑰 / 做旧边框 / 经纬网格线 / 海洋帆船·海怪插画。
- 沿路径运动的装饰 symbol（骆驼/帆船/飞鸟）。
- 背景音乐 + 翻书音效 + 与天气联动的环境音（雨声/风雪声）。
- 按真实时间的晨昏/季节调色与方向光、山间云雾。

**验收标准（通用）**
- [ ] 每个子项可单独开关，关闭后不影响主功能。
- [ ] 不引入性能反模式；开启后帧率仍可接受（必要时受性能模式约束）。

**依赖**：建议在 P0 完成后。

## 里程碑建议

1. **M1 数据与规模**：ISSUE-01、ISSUE-02 —— 打好规模化地基。
2. **M2 还原原片**：ISSUE-03、ISSUE-04、ISSUE-05、ISSUE-06、ISSUE-08 —— 质感 + 信息量 + 叙事。
3. **M3 工程与交互**：ISSUE-07、ISSUE-09。
4. **M4 氛围打磨**：ISSUE-10 子项按需。

## 给 Claude Code 的起步建议

1. 先读 `技术路径与开发交接.md` 和本文件，再 `view` `literary-map.html`，对照代码里的 ①②③④ 注释定位扩展点。
2. 从 **ISSUE-01** 开始（其他多数依赖它）。每完成一个 issue，对照"验收标准"逐条自检后再进入下一个。
3. 任何 UI/特效改动前，先确认不违反顶部的性能约定。
